Website

Website 指的是全球資訊網協定下的網站, 而全球資訊網協定是依附在 TCP/IP 與 HTTP 網路協定下的客戶端 (瀏覽器) 與伺服器端.

若利用 Github Pages 建立個人網站, 則伺服器端就是 github.io 網址中的伺服器,

Github Pages Docs: https://docs.github.com/en/enterprise-server@3.5/admin/overview/about-github-enterprise-server 

Github 帳號功能比較: https://github.com/pricing#feature-comparison

Github Pages:

  1. 由 Github 所提供的靜態網頁全球資訊網伺服器 (Static site hosting server)
  2. 可伺服 Github 倉儲中的超文件, 包括 html, css 與 Javascript 資料
  3. 帳號.github.io 是最短的網站名稱, 但要求倉儲必須命名為: 帳號.github.io
  4. 將資料轉為網站時, 必須檢查倉儲內容是否具備轉為網站的條件
  5. 網站總容量必須小於 1GB (因此倉儲中儘量不要存放大檔案, 建議將檔案放入 Onedrive 後在網站中提供連結)
  6. 網站連結總頻寬必須小於 100GB/月 (因此網站中不要放入無謂的 html 連結或設定, 意即不要直接從其他網站 copy/paste 資料, 而應提供原始文字檔後, 進入編輯器設定必要的設定與連結)
  7. 網站轉檔次數限制在每一小時 10 次 (意即每一小時的推送 (push) 次數不要超過 10 次)
  8. 使用者必須遵守 Github Terms of Service

cmsimde 利用 Python + Flask 建立動態網站, 有 login, logout, 網頁是由 Python + css + javascript 所產生. 主要功能是可以直接編輯後產生動態網頁.

cmsimde 靜態網頁資料位於 content 目錄下, 內容只有 html + css + javascript, 所有資料都是透過 WWW server 伺服, 所以安全機制在於 WWW server 與操作系統.

Nginx 建立 WWW 伺服器:

Windows:

下載 nginx-1.24.0.zip, 解開壓縮檔案後, 點選 nginx.exe 即可以下列設定檔案啟動:

    server {
        listen       80;
        server_name  192.168.1.8;
        # 防火牆必須開啟 TCP port 80 的輸入規則

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
            root   html;
            index  index.html index.htm;
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }

議題: 是否可以將 localhost 改為 Windows 所在的 IP 位址, 然後設法將個人的靜態網頁目錄置入, 利用 Nginx WWW 伺服器來 server cmsimde 靜態網頁內容?

Ubuntu:

關閉執行中的 nginx: sudo systemctl stop nginx

徹底刪除先前安裝的 nginx: sudo apt-get purge nginx nginx-common nginx-full

重新安裝 nginx: sudo apt install nginx

為每位系統用戶設定 www 伺服目錄:

sudo_user@cad2:~#sudo vi /etc/nginx/sites-available/default
# 設定讓各用戶的 public_html 目錄可以作為 nginx 伺服網頁目錄
server {
        .....
        .....
        location ~ ^/~(.+?)(/.*)?$ {
              alias /home/$1/public_html$2;
              index  index.html index.htm;
              autoindex on;
        }
# 設定完成後必須重新啟動 nginx
sudo_user@cad2:~# systemctl restart nginx

設定範例: http://cad2.cycu.org/~yen/5jgx/

請注意: 利用 public_html 設定 nginx 下的用戶倉儲, 一旦放入公開目錄後, 所有檔案都可經由全球資訊網連結取下, 其中包括 .git 目錄下的 config 設定檔案, 因此若採 git clone 將個人或分組倉儲放入各帳號 public_html 目錄下之後, 不可將維護倉儲用的 token 放入 .git 目錄下的 config 檔案中, 而必須使用 SSH 協定進行設定.

此外, 也不可利用 public_html 目錄下的倉儲建立 cmsimde 動態網站, 因為其中的 config/config 設定屬於近端帳號密碼編碼, 一旦放入 public_html, 將原先 admin 密碼修改後的密碼編碼將會暴露在公開網站中.

總結上述注意事項, 為了在多人帳號下同時利用 Python 伺服 cmsimde 動態網站 (目前採 hupper -m server.py 啟動 waitress based Flask WSGI 網站), 在個人帳號下的 cmsimde 倉儲必須分為:

  1. 以 SSH 協定在 public_html 目錄中取下個人或分組倉儲, 可以額外以目錄命名多個倉儲, 或直接將倉儲放入 public_html 進行伺服, 一旦遠端 github 倉儲更新後, 可以透過 git pull 取下最新版本.
  2. 另外以 SSH 協定, 在 public_html 的目錄中取下個人或分組倉儲, 以 hupper -m server.py 啟動後, 透過瀏覽器連結後更改管理者密碼, 之後可以利用各自分配到的 port 在瀏覽器中登入維護, 並以 acp 新增提交推送至遠端 github.com 對應倉儲, 更新後再進入 public_html, 以 git pull 取下最新版本.
