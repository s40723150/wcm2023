cad2伺服器

建立動態網站的 demo 網站: https://mdecourse-org.github.io/workstation-scrum-1 

電腦硬體:

cad2_cycu 主機板為 Acer Veriton M670G/M670 (cmd 視窗輸入 msinfo32 即可顯示出電腦硬體, 其中包括主機板型號) 出廠時間為 2009/3/20

先前安裝 Ubuntu 20.04 LTS, 於 2023/04 底更新為 Ubuntu 22.04 LTS (How?)

以下是從舊版的 LTS 升級為新版 LTS 的步驟:

先查目前 Ubuntu 操作系統的版本:

lsb_release -a

更新 LTS 版本步驟:

sudo apt update
 
sudo apt upgrade
 
sudo apt autoremove
 
sudo retoob
 
sudo do-release-upgrade

更新為 Ubuntu 22.04 後, 查詢一下 cad2_cycu 所安裝的記憶體:

grep MemTotal /proc/meminfo

發現共安裝 8GB, 也就是兩條 4GB 的記憶體, 而主機板上應該有四個記憶體的插槽, 因此若再加上相容的另外兩條 4GB, 則這一台已經服役 14 年的 Acer 電腦, 將會有 16GB 的記憶體.

電腦軟體:

操作系統: Ubuntu 22.04 LTS 生命週期顯示硬體相容性更新可至 2027 年,  且 security patch 可服務至 2033 年, 且若到了 24.04 時可以再升級至下一版的 LTS, 因此在 Power Supply 與硬碟仍然堪用的情況下, 應該還能服役幾年.

Ubuntu 已經內建基本的 Python3 與 Git, 因此只要利用 pip3 安裝 cmsimde 所需的模組, 就能夠啟動 Flask based site, 也就是所謂的動態系統.

至於目前 cmsimde 網站需要的 Python 模組有:

install flask flask_cors bs4 lxml pelican markdown pyopenssl

因此需要執行 pip3 install install flask flask_cors bs4 lxml pelican markdown pyopenssl

其中 flask 是 WWW 程式框架, flask_cors 延伸模組則可讓網站設定跨網域擷取資料, bs4 與 lxml 則用來進行網頁的分頁 (針對 config 目錄下的 content.htm 動態網站檔案), pelican 與 markdown 則用來建立 Blog 網誌文章, 最後的 pyopenssl 則當 cmsimde 網站以 localhost 啟動時 (在 Windows 執行 cms.bat 或在 Ubuntu 執行 ./cms), 自動建立 https 協定所需的 self-signed 數位簽章.

WWW server 套件: 採用 Nginx

wsgi 執行設定: letsencrypt + stunnel + waitress + hupper

建立 stunnel.conf 的程式:

# 想要透過程式方法, 建立 stunnel.conf 設定檔案, 其中分別從 9001 到 9300, 8001 到 8300
first = '''
[https]
accept = cad2.cycu.org:
'''
second = '''
connect = 127.0.0.1:
'''
third = '''cert = /etc/letsencrypt/live/cad2.cycu.org/fullchain.pem
key = /etc/letsencrypt/live/cad2.cycu.org/privkey.pem
TIMEOUTclose = 0
'''
list8 =[str(i) for i  in range(8001, 8301)]
list9=[str(i) for i  in range(9001, 9301)]

setup = ""
for i in zip(list8, list9):
    #print(i)
    print(i[0], i[1])
    setup += first.strip() + i[0] + "\n" + second.strip() + i[1] + "\n" + third

with open("stunnel.conf", "w") as f:
    f.write(setup)

使用者必須學會:

以 ssh-keygen 建立 public and private keys

使用 sftp 或 filezilla 取下 public keys 登錄到 Github.com

利用 ncat 設定 git 指令在 ssh 協定下的代理主機 (因為 cad2_cycu 只支援 IPv6 而 github.com 只支援 IPv4)

使用 chmod 將 acp 設為擁有者可以執行

了解近端啟動的 cmsimde 動態網站 port 是透過 stunnel 轉介到外部連網的 port

會使用 hupper -m server.py 啟動 cmsimde 動態網站
