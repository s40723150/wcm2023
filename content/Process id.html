Process id

In linux how to list my own process id:

To list your own process ID (PID) in Linux, you can make use of the pgrep command. Here's how you can do it:

Open a terminal and type the following command:

pgrep -u $USER

This command uses the pgrep utility to search for processes based on various criteria, such as the user who owns the process. The -u option is used to specify the username, and $USER is a shell variable that holds your username. By passing your username to pgrep, it will list the PIDs of all processes owned by you.

Alternatively, you can use the ps command with the u option to list the processes and then filter the output using grep. Here's an example:

ps u -U $USER | grep -v grep

In this command, ps u -U $USER lists all processes owned by you, and grep -v grep filters out the line containing the actual grep command itself from the output.

Both methods will provide you with a list of process IDs associated with your user account.

In linux how to list all process id that starting through python3 execution and who own those id

To list all process IDs that were started through the execution of Python 3 scripts and find out who owns those IDs, you can use the ps command in Linux along with filtering options. Here's an example:

To list all process IDs started by Python 3, you can use the following command:

ps -ef | grep python3

This command will list all processes (ps -ef) and then filter the output using grep to display only the lines containing "python3".

To also find out the user who owns each process, you can use the following command:

ps -ef | grep python3 | awk '{print $2, $1}'

The awk command is used to print the second ($2) and first ($1) fields, which represent the process ID and the user, respectively.

The output will include the process ID and the corresponding user for each Python 3 process that is currently running.

Please note that these commands will list all processes started by Python 3, regardless of whether they are still running or have already completed.

To list the process IDs started by Python 3 along with the executing directory and the corresponding Python script files, you can use the following command:

ps -ef | grep python3 | awk '{print $2, $1, $(NF-2), $(NF-1), $NF}'

Here's an explanation of the command:

  • ps -ef lists all processes.
  • grep python3 filters the output to display only the lines containing "python3".
  • awk '{print $2, $1, $(NF-2), $(NF-1), $NF}' uses awk to print the second field (process ID), first field (user), second-to-last field (executing directory), last-but-one field (script file name), and last field (command) from the filtered output.

The output will include the process ID, user, executing directory, and Python script file for each Python 3 process that is currently running.

Please note that the command assumes that the Python 3 processes are being executed directly through the python3 command. If the processes are launched with a different command or through a virtual environment, you may need to adjust the grep pattern accordingly.
