Git 指令

電腦中可以執行安裝版的 git, 也可以執行可攜版的 git.

git 安裝版: https://github.com/git-for-windows/git/releases/download/v2.40.0.windows.1/Git-2.40.0-64-bit.exe

git 可攜版: https://github.com/git-for-windows/git/releases/download/v2.40.0.windows.1/PortableGit-2.40.0-64-bit.7z.exe

git clone 指令可以透過 https 或 ssh 協定取下倉儲資料.

git clone https://github.com/帳號/倉儲.git 取下倉儲之後, 若要改版後新增提交並執行 git push, 則必須在倉儲的 .git/config 中 [remote "origin"] 下的 url 置入擁有修改 repo 權限的 token.

https://your_token@github.com/帳號/倉儲.git

假如倉儲改版會啟動 github actions, 則 token 除了 repo 權限外, 還需加上 actions 權限.

git clone 與 git push 執行若採 https 協定, 還要注意執行命令當下所使用的網路協定. 因為 github.com 目前仍只支援 IPv4 網路協定連線, 因此當使用者執行 git clone 與 git push 時若採用 IPv4 網路協定則無需在 home_ipv6 (或home_ipv4) 下的 .gitconfig 中設定 http.proxy, 但是若採用 IPv6 網路協定, 則必須設定 .gitconfig 中的 http.proxy

若使用 ssh 協定:

則在 git clone 執行之前就必須配置好以下環境:

  1. start_ipv6.bat 中必須要設定 GIT_SSH 環境變數, 且指向 Putty 中的 plink.exe:
  2. set GIT_SSH=%Disk%:\putty\plink.exe
  3. 將 OpenSSH 格式的公鑰(鎖頭)放到 Github 所提供的 public key server
  4. 在 Putty 中設定連線 session, 分別代表 session 名稱, 連線至 github.com, 是否需使用 Proxy server 以及Private key (Putty 格式的私鑰) 所在位置.
  5. git clone 時的語法: git clone git@putty_session:帳號/倉儲.git

git commit 時必須在 home_ipv6(或 home_ipv4) 目錄中的 .gitconfig 設定 user.name 與 user.email
