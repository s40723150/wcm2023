假如要選擇一項主題作為內容進行管理, 您會選擇什麼主題？

  1. 利用近端管理個人網站 - 需要能開啟動態網站的 Python 程式環境
  2. 利用 Replit 管理個人網站 - 需要建立 Replit 帳號, 並使用 Token 授權維護 Github 倉儲
  3. 利用 Stud 伺服器管理個人網站 - 需要 Stud 主機帳號, 內部與外部 port, server.py 以及 acp
  4. 利用遠端管理個人網站 - 需要透過 Github Actions 協助集結動態網站內容與轉檔

心得:

以下是讀完《1999 How to be a star engineer》後的十個重要心得報告：

  1. 學術知識、技術能力和軟實力是成為優秀工程師的基本要素，三者缺一不可。
  2. 作為工程師，不斷學習和實踐是必不可少的，以保持技能和知識的更新。
  3. 優秀的工程師必須懂得如何與人合作，透過溝通和協調，發揮團隊合作的力量。
  4. 尋找良好的導師和尋求反饋，是發展能力的重要方式。
  5. 持續反思自己的工作，不斷改進和創新，可以提高工作的效率和品質。
  6. 優秀的工程師需要具備領導能力，能夠引導團隊一起完成任務。
  7. 工程師需要擁有全球視野，關注全球最新技術和趨勢，以跟上行業的發展。
  8. 身為工程師，應該具備問題解決的能力，並且可以快速地做出決策和行動。
  9. 持續學習和提升自我價值，可以為工程師帶來更多的機會和挑戰。
  10. 最後，對工程師來說，始終保持謙虛、開放和謹慎的態度，才能達到更高的成就。

額外十個讀後心得:

  1. 為了成為優秀的工程師，需要有充分的自信和熱情，對自己的職業和行業充滿熱愛。
  2. 優秀的工程師必須具備創新和創造力，能夠提出新的想法和解決方案。
  3. 工程師需要擁有良好的時間管理能力，並能夠有效地安排工作和生活，保持工作與生活的平衡。
  4. 持續學習和研究最新技術和趨勢，可以為工程師帶來更多的競爭力和創造力。
  5. 優秀的工程師需要具備良好的人際關係和溝通能力，並且能夠有效地與不同背景的人合作。
  6. 身為工程師，需要能夠獨立思考和解決問題，並能夠快速適應不同的工作環境和工作壓力。
  7. 工程師需要懂得管理和組織，並且能夠合理地分配工作和資源，以提高工作效率和品質。
  8. 積極參與社群和行業活動，可以擴展人脈和獲得更多的機會和資源。
  9. 優秀的工程師需要擁有堅實的專業知識和技能，並能夠運用這些知識和技能解決複雜的問題。
  10. 最後，身為工程師，需要保持對工作的熱情和耐心，並且持續追求卓越，才能在職業生涯中取得成功。
