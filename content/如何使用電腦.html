如何使用電腦

  1. 儘量不要使用中文作為檔案名稱或目錄名稱 - 因為某些英語系國家開發的軟體(尤其是開源免費軟體)可能會無法使用或發生問題.
  2. 儘量不要直接將英文網頁翻譯成中文 - 因為你需要看得懂這些英文內容(即便不是現在, 但爾後作為一位工程師, 看懂各種英文技術資料是必要能力 - 你一定要懂的 5000 個英文單字). The_Oxford_5000.pdf, cambridge_english_vocabulary_list.pdf, 5000_words.zip
  3. 千萬不要忽略電腦操作過程中回應的英文訊息 - 因為一知半解與電腦溝通, 最後可能無法完成正確的操作
  4. 儘量使用 web-based 介面與虛擬主機及虛擬身分搜尋資料, 並且經常刪除暫存檔案與歷史搜尋紀錄, 因為我們不想讓搜索引擎鎖定(限制)你的未來搜尋範圍. cdwin_vdi.7z (for @nfu users only)
  5. 針對學校與住家的網路環境, 必須儘量深入研究各種使用情境與軟硬體技術規範, 以便找出最佳使用的模式或設置條件. (例如: 在學校要如何設置網路, 才能在不同協定、不同軟體套件與不同 client 端 (例如: git) 或 server 端 (例如: Nginx 或 cmsimde on Ubuntu) 的整合下達到最佳使用模式) - 例如: 透過電力系統傳輸網路的不同硬體選擇.
  6. 網路上沒有所謂的"私密空間", 當資料發布與人(家人、朋友或其他相關人)共享, 就有可能出現在其他公開網站. 
  7. 現代人可以從數位軟硬體的應用得到一定程度的好處(其中一項是模擬, 但要讓電腦模擬現實事物, 並發揮大功效, 使用者需要懂一些物理與數學), 但仍必須清楚明白虛擬世界與現實生活的差異, 取其利而避其害.

何謂可攜系統?

可以帶走的系統 (因為電腦有裝防寫, 因為設定有些複雜, 因為取下來的網站資料會逐步更新)

可以隨身執行 Python 的程式環境 (其中已經安裝執行 cmsimde 動態網站所需的 Python 模組)

可攜系統利用 Python 的特定模組來執行動態網站 (包含 flask flask_cors bs4 lxml pelican markdown leo pyopenssl 等模組)

可攜系統 -

啟動檔案 (批次檔 batch files - 將一系列的指令存在檔案中, 然後 Windows 將副檔名訂為 .bat, 只要副檔名為 .bat 且其中是可以執行的命令.)

編輯器 (SciTE), 經過設定後, 可以直接解譯 Python 程式, 並且可類解譯(即時編譯) ANSI C  程式

電腦啟動時, 有一個命令的搜尋路徑, 可以利用 path 指令取得, 在可攜啟動後所產生的 cmd 就是帶有可攜系統環境的命令列視窗, 其命令搜尋路徑就是定義在 start_ipv6.bat 的 path= 那一行

cmsimde 是甚麼? 

cmsimde 是一套利用 Python 與 Flask 所編寫的網際內容管理程式套件
